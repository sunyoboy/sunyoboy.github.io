## 产出效率修正（隐藏成本）

同样一个任务，不同模型消耗的 token 量差异巨大：

| 模型 | 同任务 token 量 | 原因 |
|------|:---:|------|
| GPT-5.5 | 基准的 **28%** | 输出极简，token 效率最高 |
| Claude Opus 4.8 | 基准 | 正常输出，设计审美好 |
| GLM-5.2 | 约 1.2-1.5 倍 | 开源模型中偏简洁 |
| 豆包 2.1 Pro | 约 1.3-1.8 倍 | 国产模型中偏正常 |
| DeepSeek V4-Pro | 约 1.2-1.5 倍 | 略偏冗长但可控 |
| Qwen3.7 Max | 基准的 **3.7 倍** | 偏好冗长输出，实际成本翻倍 |

> 🔑 GPT-5.5 虽然单价 $30 最贵，但实际单任务成本 ≈ 豆包 2.1 Pro 的 2-3 倍（而非 12 倍）。Qwen 的 ¥18/M 看起来便宜，乘以 3.7 倍后实际成本翻倍。

## 选型指南

### 按需求选

```
预算无限 + 追求极致品质  → Claude Opus 4.8     $5/$25   代码黄金标准，多文件重构
Agent 密集 + 工具调用     → GPT-5.5              $5/$30   终端自动化最强，token效率最高
中文场景 + 合规优先      → 豆包 2.1 Pro          ¥6/¥30   Coding 比肩 GPT-5.5，国产最强Coding
开源 + 自部署 + Coding   → GLM-5.2              ¥8/¥28   MIT 协议，代码能力对标 Opus 4.6-4.8
中文场景 + 性价比        → Qwen3.7 Max          ¥6/¥18   国产冠军（注意 token 偏冗长）
极致性价比 + 大批量      → DeepSeek V4-Pro      $0.44/$0.87  GPT 的 1/34 价格
最低门槛入门             → GitHub Copilot Pro    $10/月   最便宜订阅

### 工具搭配建议

大部分开发者组合两个工具：

IDE 内日常编码   → Cursor 或 Copilot（$10-20/月）
深度多文件重构    → Claude Code 或 Codex CLI（$20/月）
国产合规 + 中文   → TRAE（免费，豆包驱动）
省钱自部署        → Cline + GLM-5.2 本地 / DeepSeek V4-Pro API
```
