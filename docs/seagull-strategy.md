## 一、什么是海鸥策略

**海鸥策略（Seagull Option Strategy）** 是由 **三个到期日相同的期权** 组合而成的结构化策略。又名：兰保式期权（Rambo）、楼梯式策略（Stair Case）、三腿式组合。

> 核心思想：**用卖出的"多余"风险，来补贴买保护的成本，做到零成本甚至负成本套保。**

它可以理解为 **领口策略（Collar）的升级版**——从一路"砍价"演化而来：

```
买一个 Put 保护（贵）
  → 换成虚值 Put（便宜但保护弱了）
  → 再卖一个更低行权价的 Put 补贴成本（Put Spread）
  → 改成卖一个 Call 补贴成本（Collar / 领口策略）
  → 两个都卖：卖 Put + 卖 Call，补贴买入的保护（Seagull / 海鸥策略）
```

---

## 二、底层逻辑

海鸥策略的本质是 **做空波动率 + 方向性偏多**：

| 要素 | 逻辑 |
|------|------|
| 买 Put 保护 | 对冲下行风险 |
| 卖 Call 补贴 | 用牺牲上方超额收益换取权利金 |
| 卖 Put 补贴 | 用承担更低位置的接货义务换取权利金 |
| 三腿净成本 | 净权利金通常为零或很小，甚至为负（净收入） |

## 三、两种构建方式

### 3.1 看涨海鸥（Bullish Seagull，偏多）

适用于：**持有现货 + 预期震荡偏多不暴涨**

| 腿 | 操作 | 行权价 | 作用 |
|:---:|------|:---:|------|
| 1 | **买入 Put** | K2（平值，ATM） | 下行保护 |
| 2 | **卖出 Call** | K3（虚值，OTM，高行权价） | 补贴权利金，封顶收益 |
| 3 | **卖出 Put** | K1（虚值，OTM，低行权价） | 进一步补贴，换取接货义务 |

**到期损益：**

```
K1 < K2 < K3

X ≤ K1  → 亏损加大（卖出的 Put 被行权）
K1 < X ≤ K2  → Put 保护生效，获得保护收益
K2 < X < K3  → 上涨获利
X ≥ K3  → 收益封顶（卖出 Call 被行权）
```

### 3.2 看跌海鸥（Bearish Seagull，偏空）

适用于：**持有现货 + 预期震荡偏弱不暴跌**

| 腿 | 操作 | 行权价 | 作用 |
|:---:|------|:---:|------|
| 1 | **买入 Call** | K2（平值） | 上行保护（防止卖飞） |
| 2 | **卖出 Put** | K1（虚值，低行权价） | 补贴权利金，承接补仓义务 |
| 3 | **卖出 Call** | K3（虚值，高行权价） | 进一步补贴，封顶收益 |

## 四、A股实战：50ETF/300ETF期权

### 4.1 A股可用的场内期权

| 品种 | 合约单位 | 交易场所 |
|------|:---:|------|
| 上证50ETF（510050）期权 | 10,000份/张 | 上交所 |
| 沪深300ETF（510300）期权 | 10,000份/张 | 上交所/深交所 |
| 中证500ETF（510500）期权 | 10,000份/张 | 上交所/深交所 |
| 科创50ETF（588000）期权 | 10,000份/张 | 上交所 |
| 创业板ETF（159915）期权 | 10,000份/张 | 深交所 |

> 📌 50ETF 和 300ETF 期权流动性最好，是海鸥策略的首选。

### 4.2 看涨海鸥实战案例（以 50ETF 为例）

**假设当前场景：**

- 50ETF 现价：**2.800 元/份**
- 持有现货：50,000 份（市值约 14 万元）
- 到期月：近月合约（约 4 周后到期）
- 预期：震荡偏多，跌不深也涨不高

**构建看涨海鸥（1 套合约对应 10,000 份现货 = 1 张）：**

| 腿 | 操作 | 行权价 | 权利金（预估） | 说明 |
|:---:|------|:---:|:---:|------|
| 1 | **买入 Put** | 2.800（平值） | -0.050 元/份 | 买入下行保护 |
| 2 | **卖出 Call** | 2.900（虚值 3.6%） | +0.020 元/份 | 如果涨超 2.900，别人有权买走我的现货 |
| 3 | **卖出 Put** | 2.650（虚值 -5.4%） | +0.015 元/份 | 如果跌破 2.650，我有义务加仓接货 |
| **净权利金** | | | **-0.015** 元/份 | 极低成本！仅需净支出 150 元/套 |

### 4.3 到期损益分析

| 到期价格 | 结果 |
|------|------|
| **50ETF 大涨至 3.000** | 卖出的 Call 被行权，收益封顶在 2.900，错过 2.900→3.000 的涨幅 |
| **50ETF 小涨至 2.850** | 现货盈利 + 无期权被行权，全部收入口袋 |
| **50ETF 横盘在 2.800** | 现货不赚不亏，期权净支出 150 元 |
| **50ETF 小跌至 2.700** | Put 保护生效！卖出的 2.650 Put 未被行权，2.800 Put 有保护利润 |
| **50ETF 大跌至 2.500** | 2.800 Put 全额保护。但 2.650 Put 被行权——你有义务在 2.650 接货 |

## 五、收益风险全景

### 损益图（看涨海鸥）

```
收益
  ↑
  │           ╱‾‾‾‾‾‾‾‾‾‾
  │         ╱              ← 收益封顶（超过 K3 不再赚）
  │       ╱
  │     ╱
  ├───╱ ← 盈亏平衡点
  │ ╱
  │╱
──┼──────────────→ 到期价格
  │╲
  │ ╲
  │  ╲ ← K1 以下加速亏损
  │
  ↓
```

| 价格区间 | 结果 |
|------|------|
| **> K3** | 收益封顶 |
| **K2 ~ K3** | 上涨获利 |
| **K1 ~ K2** | Put 保护生效 |
| **< K1** | 亏损扩大 |

### 盈亏平衡点

```
盈亏平衡点 = K2 - 净权利金
```

如果买入 Put 2.800 花费 500，卖出 Call 收入 200，卖出 Put 收入 150，则：

> 净权利金 = 500 - 200 - 150 = **150 元/套（支出）**
> 盈亏平衡点 = 2.800 - 0.015 = **2.785**

50ETF 跌到 2.785 以下才开始亏损。

## 六、适用场景

| 场景 | 为什么海鸥策略合适 |
|------|------|
| **FOMC 前夕** | 预期振幅收窄，做空波动率赚钱 |
| **4000 点反复拉锯** | 上有压力下有支撑，区间震荡 |
| **节前/长假前** | 交投清淡，方向不明 |
| **持仓过节** | 低成本甚至零成本买保护 |
| **做T + 期权增强** | 底仓不动，用期权覆盖做T的机会成本 |

### 反制指标：当下不适合海鸥策略的信号

- VIX / 波动率极度压缩 → 卖权收到权利金太少，不划算
- 单边趋势确立（均线多头/空头排列）→ 收益封顶会错失利润
- 重大事件前后 1-2 天（如 FOMC 决议日）→ 波动率可能飙涨，卖权风险大

## 七、操作流程

| 步骤 | 动作 |
|:---:|------|
| 1 | 确认方向：震荡偏多还是偏空？ |
| 2 | 选合约月：近月（流动性好）还是次月（时间更长）？ |
| 3 | 定行权价：K2 平值，K1/K3 虚值多少合适？ |
| 4 | 计算净权利金：是否零成本/低成本？ |
| 5 | **一次性下三腿组合单**（不要分三次下，手续费更高且有执行风险） |
| 6 | 持仓期间：关注是否接近 K1 或 K3，准备应对 |
| 7 | 到期/平仓：到期前可提前平仓，不用等到交割 |

### A股期权交易门槛

- **🔴 50 万资产硬门槛**（20日均资产 ≥ 50万，才能开通期权账户）
- 通过知识测试
- 模拟交易经历
- 保证金制度（裸卖 Put 需要保证金）

> ⚠️ **如果你的总资产不足 50 万，A 股场内期权无法操作。** 海鸥策略仅限达标账户。详见下方替代方案。

## 八、风险提示

| 风险 | 说明 |
|------|------|
| 收益封顶 | 大涨时无法获得全部利润，心理上可能"踏空" |
| 下方接货 | 跌破 K1 时有义务加仓，需要足够资金 |
| 波动率风险 | 隐含波动率突然飙升时，卖出期权亏损可能远超收到的小额权利金 |
| 流动性风险 | 非主力合约流动性差，平仓滑点大 |
| 操作门槛 | 需要期权交易权限 + 50 万资产门槛 |

## 九、不足 50 万的替代方案

如果达不到 A 股期权门槛，海鸥策略的**底层逻辑**仍然可用——用其他工具模拟类似效果：

| 海鸥策略要素 | 替代方案（无需期权） |
|------|------|
| **买 Put 保护** | 预留现金（如仓位从 60% 降到 40%），用**现金 = Put 保护** |
| **卖 Call 收益封顶** | **预设止盈单**——价格涨到目标位自动卖出，效果等同卖 Call |
| **卖 Put 接货义务** | **预设条件单**——跌到目标位自动买入，效果等同卖 Put |
| **三腿组合** | **网格交易**——在区间内反复高抛低吸，天然做空波动率 |
| **零成本** | 现金管理 + 做T，用逆回购和日内波段收益覆盖机会成本 |

### 实用替代：网格交易 + 做T 组合

```
震荡区间：上证 3900 ↔ 4033（50ETF 约 2.65 ↔ 2.90）

网格设置：
  2.90 → 卖出 20% 仓位（等同于卖 Call 被行权）
  2.82 → 卖出 10%
  2.78 → 不动
  2.72 → 买入 10%
  2.65 → 买入 20%（等同于卖 Put 被行权）

做T增强：
  日内 A/B 级信号出现时，用底仓做T赚额外 0.5%-1%
```

> 🔑 **逻辑相通，工具不同。** 海鸥策略的核心是"区间内获利、区间外有限风险"，这个逻辑不用期权也能执行——用**仓位管理 + 网格 + 做T**替代三腿期权。效果不如期权精确，但门槛为零。

---

> 🔑 **海鸥策略不是万能药。它是震荡市的最佳武器，但单边市场里会被打脸。核心在于择时：只在预期波动率下降、方向不明朗时使用。**
