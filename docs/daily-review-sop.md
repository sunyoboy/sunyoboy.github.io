# 每日复盘 SOP · 标准化操作流程

> 一条流水线·七个步骤·60分钟内完成·每个步骤有检查点

---

## 流水线总览

```
Step 1         Step 2        Step 3        Step 4        Step 5        Step 6        Step 7
拉数据         核对          诊断          写复盘         计划          检查          推送
(5min)        (2min)       (10min)       (20min)       (10min)       (5min)        (3min)
```

---

## Step 1：拉数据（15:05 执行·等收盘价稳定）

```bash
cd $KNOWINGDOING_HOME
python3 scripts/fetch-market-data.py     # 7大指数 + 6个标的
python3 scripts/ma5-deviation.py         # 持仓三维诊断
python3 scripts/shenwan-monitor.py       # 31行业全景
```

**检查点：** 三脚本全部 ✅ 才能继续。任何一个 ❌ → 重新跑。

---

## Step 2：核对（防串价·必做）

> 教训来源：W28 串价 + 7/17 用 14:27 数据当收盘

| 检查项 | 方法 |
|------|------|
| 时间校验 | 确认抓取时间 ≥ 15:00。如果是盘中快照 → 标注"⚠️ 盘中数据"→ 收盘后用华泰 API 订正 |
| 价格校验 | 对照价格速查表逐只核对（见 CLAUDE.md 数据核对清单） |
| 异常检测 | 偏离度 > ±15% 的标的 → 手工查华泰 API 确认 |

**检查点：** 核对不通过 → 不开写复盘。

---

## Step 3：诊断（按决策矩阵·不跳项）

### 3.1 MA5 偏离度 → 操作指令

| 信号 | 指令 | 优先级 |
|------|------|:--:|
| 死叉▼ + 偏离 < -5% + 偏离扩大 | **清仓** | 🔴 当天执行 |
| 死叉▼ + 偏离 < -3% | **减仓 1/2** | 🔴 当天执行 |
| 金叉▲ + 偏离 < -3% + 偏离收窄 | **低吸**（按计划挂单） | 🟡 条件触发 |
| 金叉▲ + 偏离 > +5% + 偏离收窄 | **减仓 1/3 锁利** | 🟡 条件触发 |
| 贴合 ±2% | **持有/观察** | 🟢 |
| 死叉▼ + 偏离 < -8% + 连续 3 日 | **市价清仓·不等反弹** | 🔴🚨 |

### 3.2 止损止盈检查

| 触发条件 | 指令 |
|------|------|
| 宽基 ETF 跌超 -3% | 卖出 |
| 行业 ETF 跌超 -5% 或持有 > 20 日未盈利 | 卖出 |
| 个股跌超 -8% 或持有 > 10 日未盈利 | 卖出 |
| 正偏 > +5% + 偏离收窄 | 减 1/2 锁利 |
| 三连大阳线 | 减仓 |

### 3.3 买入前必问（缺一不买）

```
① PE < 20（ETF 看 PE 分位 < 20%）？
② 近 10 日主力净流入？
③ 公司没发炒作风险/减持公告？
④ 入场价 + 止损价 + 仓位 = 三个都填好了？
```

---

## Step 4：写复盘（按模板·不跳项）

模板在 `review/YYYY/MM/YYYY-MM-DD.md`，必须包含：

```
□ 指数表现（7 个指数·含四天累计）
□ 全市场统计（涨跌比·成交额·主力/北向/融资）
□ 全天走势分段（至少 3 阶段·含时间节点）
□ 申万行业（领涨 5·领跌 5·涨跌比·极端信号）
□ 重点标的（持仓逐只·含 MA5 偏离+均线+操作建议）
□ 盘面特征（3-5 条·有数据支撑）
□ 今日驱动（按权重排序·每条一句话说清因果链）
□ 技术面（每个指数一句话）
□ 后市关键（3-5 个待验证信号）
□ 做了什么（含未操作的理由）
□ 持仓盈亏（三账户汇总）
□ 学到了什么（至少 1 条·不流水账）
□ 明日计划（具体标的+价格+条件+不做清单）
□ 如果当日有全球市场联动 → 加入隔夜美股分析
```

---

## Step 5：写计划（盘前挂单·情景推演）

```
□ 🔴 盘前挂单（市价·不等·不找理由）
□ 🅰️ 情景A（低开/继续跌）
□ 🅱️ 情景B（高开/反弹）
□ ⚪ 不做清单
□ 👀 核心观察清单（优先级排序）
```

---

## Step 6：检查（自检清单·提交前过一遍）

| 检查项 | ✅ |
|------|:--:|
| 数据是收盘数据（≥15:00）？ | □ |
| 14:27 旧数据已替换为收盘数据？ | □ |
| 价格在合理区间内？ | □ |
| 持仓诊断有 MA5 偏离度 + 操作建议？ | □ |
| 明日计划有具体标的+价格+条件？ | □ |
| 学到了什么不是流水账？ | □ |
| 新增文件 catalog.md + config.mjs 已更新？ | □ |
| npm run build 通过？ | □ |

---

## Step 7：推送

```bash
npm run build && git add -A && git commit -m "feat(review): YYYY-MM-DD ..." && git push
```

---

## 异常处理

| 异常 | 处理 |
|------|------|
| 抓取的数据是盘中/昨天的 | 标注"⚠️ 盘中数据"→ 收盘后用华泰 API 订正 |
| 华泰 API 超时 | retry 1 次·仍超时 → 用脚本数据·标注来源 |
| MA5 脚本输出异常值 | 手工查华泰 API 确认单只标的 |
| 有紧急操作信号（冰点/止损触发） | **当天执行·不等复盘写完** |
