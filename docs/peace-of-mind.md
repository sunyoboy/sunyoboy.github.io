## 一、先认清一个事实

> **糟心的人和事永远存在，这是矛盾的普遍性决定的。** 没有矛盾就没有世界。你不可能消灭它们，只能改变你和它们的关系。

心态好的本质不是"没有烦心事"，而是：**烦心事来了，它只影响你的情绪层，不影响你的决策层。**

```
没有操作系统的人：糟心事 → 情绪失控 → 错误决策 → 更多糟心事（恶性循环）
有操作系统的人：糟心事 → 情绪波动 → 系统兜底 → 按规则执行 → 不影响大局
```

---

## 二、四个步骤

### 第一步：区分可控与不可控

| 你不可控的 | 你可控的 |
|-----------|---------|
| 别人说什么、做什么 | 你怎么回应 |
| 市场怎么走 | 你怎么操作 |
| 已经发生的事 | 你赋予它什么意义 |
| 运气 | 策略 |

> 把 90% 的注意力放在右栏。左栏的事，接受它发生，然后问自己："我现在能做什么？"

### 第二步：回到主要矛盾

问自己一个问题：**这件事，在我要达成的主要目标中排第几？**

| 如果它的排名是... | 处理方式 |
|------|------|
| 和主要目标无关 | 不值得花情绪。认了，过了 |
| 影响主要目标但可控 | 立刻行动解决，不拖延 |
| 影响主要目标但不可控 | 接受最坏结果，调整计划 |

> 矛盾论的核心：抓主要矛盾。**糟心的事十有八九是次要矛盾，不值得消耗战略资源。**

### 第三步：拉长时间尺度

问自己三个问题：

```
10 分钟后，我还会为这件事生气吗？
10 个月后，这件事还重要吗？
10 年后，我还记得这件事吗？
```

如果一个东西过不了"10 个月后"这一关，它就不值得你现在投入超过 10 分钟的情绪。大多数糟心事，10 天后就想不起来了。

### 第四步：转化为系统规则

**每一次糟心的经历，都是操作系统的一次 bug report。**

| 糟心事 | 提炼规则 |
|--------|---------|
| 被人放鸽子 | 以后重要约定提前 1 天确认 |
| 被恶意评价 | 只接受有建设性的批评，其余无视 |
| 交易亏损后心态崩了 | 亏损达到日限额自动停止交易 |
| 和人争执浪费半天 | 争论不超过 3 轮，第 4 轮说"我理解你的观点"然后结束 |

> 这样，每次糟心事都在让你的系统更强。**坏事变成了补丁。**

## 三、关联框架

| 框架 | 应用 |
|------|------|
| **矛盾论** | 区分主要矛盾（你的目标）和次要矛盾（糟心事）。不要用大炮打蚊子 |
| **实践论** | 心态是练出来的，不是想出来的。每经历一次糟心事，就是一次练习机会 |
| **岿然不动** | 敌军围困万千重，我自岿然不动。外界喧嚣是敌军的呐喊，你的内核是那座山 |
| **反脆弱** | 不是从糟心事中恢复，而是从糟心事中获益——每次提炼一条规则 |
| **操作系统** | 情绪会波动，系统不会。有系统兜底，心态崩了也不影响决策层 |

---

## 四、一句话

> **糟心事是操作系统收到的 bug report。不回应情绪，只回应规则。**

Sources: 矛盾论、实践论、斯多葛哲学（控制的二分法）、反脆弱
