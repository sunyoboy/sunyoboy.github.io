## 一、什么是个人操作系统

```
计算机 OS                         个人 OS
─────────                        ─────────
管理 CPU、内存、IO              管理时间、注意力、精力
调度多任务                      平衡工作/投资/学习/生活
崩溃后恢复                      亏损/挫折后恢复节奏
底层不变，上层应用可换            框架不变，策略/方法可迭代
```

**个人操作系统**是一套内化的**规则、习惯、流程和决策框架**的集合。它不替你做具体的事，而是确保你做每一件事的方式是**可重复、可优化、抗干扰**的。

> 外界信息是无限的，操作系统帮你**过滤噪音 → 做出决策 → 持续迭代**。没有它，每天的反应都是随机的；有了它，每天的反应都是系统驱动的。

## 二、为什么需要它

| 没有操作系统 | 有操作系统 |
|-------------|-----------|
| 凭感觉决定今天做什么 | OKR → 周计划 → 日执行，自动排优先级 |
| 被市场情绪牵着走 | 交易纪律预先设定，触发即执行 |
| 同样的错误反复犯 | 复盘 → 问题清单 → 知识库 → 下次避开 |
| 学了很多用不出来 | 方法论沉淀为规则，规则驱动行动 |
| 连续亏损后崩溃 | 风控系统兜底，亏损不伤本金和心态 |

**操作系统 = 把"靠意志力"变成"靠系统"。** 意志力是有限的，系统是自动的。

## 三、四层架构

每一层解决一个核心问题：

```
┌─────────────────────────────────┐
│  第 4 层 · 回顾层               │  ← 我在正确的路上吗？
│  周报/月回顾/年度复盘            │
├─────────────────────────────────┤
│  第 3 层 · 执行层               │  ← 今天做什么？
│  日复盘/交易操作/学习实践         │
├─────────────────────────────────┤
│  第 2 层 · 规划层               │  ← 为什么做这个？
│  OKR/月目标/周计划               │
├─────────────────────────────────┤
│  第 1 层 · 捕获层               │  ← 什么进来了？
│  信息/想法/问题/任务              │
└─────────────────────────────────┘
         ↑
    信条（精神内核）
```

| 层 | 功能 | 你的工具 |
|------|------|------|
| 捕获 | 所有信息、想法、问题不遗漏 | 问题清单 |
| 规划 | 把想法变成可执行的目标 | 年度 OKR → 月目标 → 周计划 |
| 执行 | 按计划行动，记录过程 | 日复盘 + 交易执行 |
| 回顾 | 对比实际 vs 目标，找偏差原因 | 周报/月回顾/年度复盘 |

## 四、五大子系统

### 1. 目标系统 — 回答"去哪"

| 方法 | 说明 |
|------|------|
| OKR | 年目标(O) → 关键结果(KR) → 季度里程碑 → 月/周任务 |
| 反推法 | 从想要的终点反推，每年/每月/每周该做什么 |
| 最少可行目标 | 不要列 20 个目标，一次聚焦 ≤3 个 |

### 2. 知识系统 — 回答"怎么积累"

| 方法 | 说明 |
|------|------|
| 费曼学习法 | 写出来、讲给别人听，看能不能讲清楚 |
| 卡片盒笔记法 | 用自己的话写笔记 → 建立链接 → 形成知识网络 |
| 主题阅读 | 围绕一个问题集中读 3-5 本书/资料 |

### 3. 执行系统 — 回答"什么时候做"

| 方法 | 说明 |
|------|------|
| GTD | 收集 → 整理 → 组织 → 回顾 → 执行 |
| 最小行动法 | 如果一件事 2 分钟内能做完，立刻做 |
| 番茄钟 | 25 分钟专注一件事，不停切换 |

### 4. 复盘系统 — 回答"怎样做得更好"

| 方法 | 说明 |
|------|------|
| PDCA | Plan → Do → Check → Act，螺旋上升 |
| GRAI | Goal（目标）→ Result（结果）→ Analysis（分析）→ Insight（洞察） |
| 5 Whys | 连续问 5 个"为什么"，找到根因 |

### 5. 决策系统 — 回答"怎么选"

| 方法 | 说明 |
|------|------|
| 第一性原理 | 拆到不能再拆的基本事实，从那里重新推理 |
| 逆向思考 | 不问"怎么成功"，问"怎么失败，然后避开" |
| 三筐分类 | Yes / No / Too Hard。大部分事属于第三筐，直接放弃 |
| 能力圈 | 只在看得懂的领域下注，不知道比知道更重要 |

## 五、关键思维模型

来自查理·芒格等人的经典智慧：

### 多元思维模型

不要手里只有一把锤子，看什么问题都是钉子。从多学科提取核心模型交叉使用：

| 学科 | 模型 | 应用 |
|------|------|------|
| 心理学 | 损失厌恶 | 为什么止损那么难（亏损的痛苦是盈利的 2 倍） |
| 物理学 | 临界点 | 趋势反转往往在一个关键事件后加速 |
| 生物学 | 进化论 | 策略也要优胜劣汰，不适应的淘汰，好的保留 |
| 数学 | 复利 | 每天进步 1%，一年后是 37.8 倍 |
| 经济学 | 机会成本 | 做这件事，意味着放弃了什么 |

### 反脆弱

> 不是抵抗冲击，而是**从冲击中获益**。

- 脆弱：压力来了就崩（重仓单票 + 不止损）
- 强韧：压力来了扛得住（分散 + 有止损）
- **反脆弱**：压力来了变得更强（每次亏损都变成一条规则写入系统）

## 六、如何从零搭建

### 第一步：定义信条（1 小时）

回答：**什么是我绝不动摇的？**

```
你的信条：
├── 不抛弃，不放弃
├── 三个不相信
├── 岿然不动
└── ...
```

信条是操作系统的 root 权限——它决定什么可以做、什么绝对不能做。

### 第二步：建立捕获层（即刻）

选一个地方，所有想法、任务、问题全部丢进去，不遗漏。你已经有了：**问题清单**。

### 第三步：设定 90 天目标（2 小时）

- 选 ≤3 个目标（多了等于没有）
- 每个目标 ≤3 个可量化的 KR
- 拆解到每月/每周

你已经有了：**年度 OKR**。

### 第四步：建立执行节奏（每日 15 分钟）

| 时间 | 动作 |
|------|------|
| 盘前 5 分钟 | 看日历，定当日策略 |
| 盘后 10 分钟 | 写日复盘 |

### 第五步：建立回顾节奏（每周 30 分钟 + 每月 1 小时）

- 周末：PDCA 周报
- 月末：月回顾 + 更新仪表盘

你已经有了模板。

### 第六步：沉淀规则（持续）

每次犯错 → 提炼一条规则 → 写入系统 → 永远不再犯同样的错。

```
犯错 → 找到根因（5 Whys）→ 写一条规则 → 加入知识库
```

## 七、与本项目的对应

你的 KnowingDoing 项目已经具备个人操作系统的骨架：

| OS 层 | 系统 | 你的工具 |
|------|------|------|
| 内核 | 信条 | 不抛弃不放弃 / 三个不相信 / 岿然不动 |
| 调度器 | 方法论 | PDCA × 知行合一 |
| 目标管理 | 目标系统 | 年度 OKR |
| 输入捕获 | 问题清单 | questions/ |
| 执行记录 | 复盘系统 | review/ 日复盘 |
| 检查回顾 | 周报 + 月回顾 | growth/ 周报模板 + 月回顾 |
| 知识沉淀 | 知识库 | docs/ |
| 决策框架 | 矛盾论 + 实践论 | docs/contradiction + practice-theory |

> 你已经在做的事情，用"操作系统"的视角重新看，就是一个完整系统。

## 八、常见失败模式

| 模式 | 表现 | 解法 |
|------|------|------|
| **工具沉迷** | 花大量时间搭 Notion/仪表盘，不实际执行 | 工具够用就行，先跑通流程 |
| **目标过载** | 同时追 10 个目标，全部浅尝辄止 | 一次 ≤3 个 |
| **复盘断层** | 写了日复盘，但从不回看和提炼规则 | 月末必须回看、提炼 |
| **规则空洞** | 定了规则但不执行（"止损设了但总挪"） | 把止损变成自动条件单 |
| **系统漂移** | 慢慢不写了，系统名存实亡 | 把系统做得足够轻，轻到即使状态不好也能坚持 |

> **最好的操作系统不是功能最全的，是你能长期坚持的。**
