# 预期与现实的差距

> 事情的发展很少按你的预期走。关键不是让事情按预期走，而是**预期偏离后，你怎么应对**。

## 一、为什么预期经常落空

三个根本原因：

| 原因 | 说明 |
|------|------|
| **信息不完备** | 你永远无法掌握所有变量。预期是基于有限信息的推测 |
| **系统有惯性** | 人和事都有自己的运行逻辑，不会因为你设了预期就改变 |
| **你在成长** | 过去的经验形成的预期，面对的是新的你和新环境 |

> 实践论告诉我们：预期 = 感性认识。它需要经过实践的反复检验和修正，才能接近客观规律。

所以**预期落空不是异常，是常态。** 需要解决的不是"怎么让预期永远准确"，而是"偏离了怎么办"。

## 二、度的判断：什么时候该管，什么时候不该管

你的直觉是对的：**只要不影响目标实现，就可以放手。**

关键在于——你得先知道"什么影响目标，什么不影响"。这需要一套刻度：

| 偏离等级 | 定义 | 举例 | 响应 |
|:---:|------|------|------|
| 🟢 **一级** | 和预期不同，但**不影响任何目标** | 约好 2 点见面对方 2:15 才到 | **不管。** 接受。连提都不用提 |
| 🟡 **二级** | 影响了执行路径，但**不影响最终结果** | 项目某个环节比预期多花了两天，但总工期有余量 | **记录但不干预。** 留到复盘时分析 |
| 🟠 **三级** | 影响了阶段性结果，**可能威胁最终目标** | 本月收益率偏离目标 30% | **调整执行方案。** 加资源 / 换方法 / 修正路径 |
| 🔴 **四级** | **直接威胁核心目标** | 交易连续亏损触及月风控线 | **立即干预。** 停止交易、全面复盘、调整策略 |

**度的核心就是：只在 🟠 和 🔴 时出手。🟢 和 🟡 的事，观察即可。**

## 三、三步处理框架

### 第一步：判断偏离等级

偏离发生了，先问三个问题：

```
① 这件事偏离了我的预期，它属于哪个等级？
   （对照上面四个等级）

② 它影响的是主要目标还是次要目标？
   （矛盾论：主要矛盾 vs 次要矛盾）

③ 如果不干预，最坏结果是什么？我能承受吗？
   （逆向思考）
```

大多数偏离是 🟢 或 🟡——和你的核心目标没有直接关系，只是因为"和想的不一样"而让你不舒服。这种不舒服，不值得转化为行动。

### 第二步：按等级响应

| 等级 | 动作 | 原则 |
|:---:|------|------|
| 🟢 | **什么都不做。** 心里过一下就行 | 不出手才是高手 |
| 🟡 | **记一笔。** 复盘时分析原因，但不中断执行 | 让子弹飞一会儿 |
| 🟠 | **调整路径。** 目标不变，换条路走 | 灵活比坚持更重要 |
| 🔴 | **立即干预。** 暂停、复盘、重新规划 | 刹车比油门更重要 |

> 大多数人的问题不是不行动，而是**对 🟢 🟡 过度反应**，把能量消耗在了不该消耗的地方。

### 第三步：更新预期模型

每次偏离都是一次校准：

```
预期偏离 → 分析原因 → 更新认知 → 下次预期更准
```

> 实践论：认识从实践始，经过实践得到了理论的认识，还须再回到实践中去。每一次偏离→修正，你的预期模型就迭代一次。

## 四、关联框架

| 框架 | 应用 |
|------|------|
| **矛盾论** | 区分主要矛盾和次要矛盾。偏离是否影响主要矛盾？不影响的，放手 |
| **实践论** | 预期 = 感性认识，需要通过实践反复修正。偏离是修正的信号，不是失败 |
| **操作系统** | 系统要设置"偏离容忍度"。在容忍度内的，系统自动忽略；超出的，触发干预 |
| **岿然不动** | 敌人围困万千重（偏离此起彼伏），我自岿然不动（内核稳定，只响应四级偏离） |
| **三筐分类** | 大部分偏离属于"Too Hard"筐——不必搞明白为什么，接受它发生了就行 |

---

## 五、一句话

> **预期偏离是常态。只在偏离威胁到核心目标时出手。其他时候，接受、记录、继续前进。**
>
> **度就在：知道什么值得管，什么不值得管。**
