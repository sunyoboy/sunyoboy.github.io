## 理论基础

### 矛盾论的视角

> 引自 [[contradiction]]

任何事物内部都存在矛盾。对自己而言：

- **优势 = 主要矛盾的主要方面**（当前阶段起主导作用的能力）
- **劣势 = 矛盾的次要方面**（当前阶段被压制但可能转化的部分）
- 优势和劣势**互相转化**——优势用过度变劣势（自信→自负），劣势被刻意训练变优势

自我认知不是"找到答案然后永远不变"——是不断跟踪矛盾的转化过程。

### 实践论的视角

> 引自 [[practice-theory]]

```
实践 → 认知 → 再实践 → 再认知 （循环往复）

打羽毛球 → 发现体能下降 → 这是劣势信号
以为自己能熬夜 → 试了一次崩溃 → 修正认知
写了策略没人买 → 发现"擅长"和"有人付费"是两回事
```

**认知只能从实践中来，不能从冥想中来。** 想知道自己擅长什么——去做。想知道自己不擅长什么——也去做。

### PDCA × 知行合一

> 引自 [[方法论]]

```
Plan   → 设定假设："我擅长 X"
Do     → 在真实场景中验证
Check  → 收集外部反馈（不是自我感觉）
Act    → 修正假设 → 回到 Plan
```

这个循环每跑完一次，自我认知就往深一层。

## 优势/劣势审计体系

### A/B 分类法

不是所有劣势都需要被克服。先分类：

```
A类：绕不开的 → 补到及格线（60分）
  ├── 投资风控纪律       → 必须 60 分
  ├── 家庭财务管理       → 必须 60 分
  ├── 公众表达           → 如果要做内容输出，必须 60 分
  └── 身体健康           → 必须 60 分（根本，没有身体什么都别谈）

B类：可以绕开的 → 直接放弃，找别人代替
  ├── 硬件维修           → 不学
  ├── 羽毛球教学         → 不碰
  ├── UI 设计（非核心）  → 用模板/AI 代替
  └── 任何与飞轮无关的手艺 → 不碰，不犹豫
```

> ⚠️ 大多数人的问题不是"不知道自己劣势是什么"，而是**在 B 类劣势上浪费了太多时间试图补成优势**。

| 能力域 | 自我评价 | 外部反馈 | 付费验证 | 结论 | 行动 |

| Python + AI 工具链 | 熟练 | ✅ 知识库一个月建成 | — | 🟢 优势 | 深化，做产品 |

## 五个操作原则

### 1. 外部反馈 > 自我评价

你自己觉得擅长什么都不算——问三个信得过的人：

> "你觉得我最强的能力是什么？"
> "你觉得我最弱的短板是什么？"

如果你听到的答案和自己的判断一致 → 认知准确。不一致 → 问具体事例，用事实校准自我认知。

### 2. 用钱投票

你真正擅长的事 = **别人愿意为你付费的事**。

```
写了策略 → 有人买吗？     ← 付费率是优势的量化指标
发了文章 → 有人订阅吗？   ← 付费率是优势的量化指标
做了工具 → 有人付费吗？   ← 付费率是优势的量化指标
```

没有付费验证的"擅长"，可能只是"自己觉得擅长"。

### 3. 记录"心流时刻"和"煎熬时刻"

| 记录 | 含义 | 最近例子 |
|------|------|------|
| 做这件事忘了时间 | 可能是你的优势区 | 写复盘、调脚本 |
| 做这件事每一分钟都在煎熬 | 大概率是劣势区，或该交给别人 | — |
| 别人做很痛苦，你做很轻松 | 这是你被低估的优势 | AI工具链的使用 |
| 别人轻松超越你，你怎么努力都不行 | 这可能不是你的赛道 | — |

### 4. 区分"财务增长点"和"生活必需品"

| 类型 | 例子 | 能变现吗 | 应该变现吗 |
|------|------|:---:|:---:|
| 🟢 财务增长点 | Python、AI、投资经验、写作 | ✅ | ✅ 全力变现 |
| 🔵 生活必需品 | 运动、睡眠、家庭、朋友 | ❌ | ❌ 不要异化 |
| 🔴 纯耗散 | 硬件维修、羽毛教学 | ❌（于你而言） | ❌ 直接放弃 |

> 今天上午打羽毛球 = 🔵 生活必需品。它让你 30 年后还能站着交易，不需要赚今天的面包。**不要什么都想着变现——有些东西的价值恰恰在于它不是生意。**

### 5. 定期审计

每个月做一次（见下方模板）。核心三问：

```
本月发现了自己什么新优势？（之前不知道的）
  →

本月发现什么我确实做不好？（接受现实）
  →

有什么事情我以前觉得是劣势，现在看可能是优势？（被低估的）
  →
```

## 财务增长点评判标准

当你产生"某某事情能不能成为一个新的财务增长点"时，用这张表检验：

| 检验维度 | 问题 | 通过条件 |
|------|------|------|
| **技能交集** | 与现有飞轮（AI×投资×Python×写作）有交集吗？ | 至少 2 项有交集 |
| **规模化** | 一份时间能不能卖给多个人？ | 能（代码/内容/策略） |
| **地理约束** | 需要物理在场吗？ | 不需要（线上即可） |
| **时间弹性** | 碎片时间能做吗？ | 能（不与孕期/育儿冲突） |
| **竞争壁垒** | 你的独特优势是什么？ | 有复合能力，别人难以复制 |
| **付费意愿** | 有人已经在为类似东西付费吗？ | 有（不需要教育市场） |

### 检验案例

**羽毛球教学**：

| 检验维度 | 结果 |
|------|:---:|
| 技能交集 | ❌ 与飞轮零交集 |
| 规模化 | ❌ 计件制，教一小时赚一小时 |
| 地理约束 | ❌ 必须在球馆里 |
| 时间弹性 | ❌ 固定时段 |
| 竞争壁垒 | ❌ 持证教练遍地都是 |
| 付费意愿 | ❌（你没教练证，家长不认） |
| **判定** | 🔴 **不是财务增长点** |

**AI 增强的付费复盘**：

| 检验维度 | 结果 |
|------|:---:|
| 技能交集 | ✅ 投资 + AI + 写作 三项重叠 |
| 规模化 | ✅ 写一次，无限分发 |
| 地理约束 | ✅ 线上即可 |
| 时间弹性 | ✅ 碎片可做 |
| 竞争壁垒 | ✅ AI 工具链 + 实盘经验，复合门槛高 |
| 付费意愿 | ✅ 知识星球/公众号已有大量类似案例 |
| **判定** | 🟢 **是财务增长点** |

## 打破公开发布卡点

> "公开发布是卡点——内容一直在产出，但从来没有推向外部。"

### 不是能力问题

你已经在写了一整个知识库——几十篇复盘、十几篇深度文档、量化策略、ETF 做T 指南、AI 模型排名——但一篇都没公开发过。恰恰证明**不是能力问题，是心理门坎**。

### 卡住的真正原因

```
"写得还不够好"         ← 完美主义
       +
"发了没人看怎么办"      ← 暴露恐惧
       +
"我又不是专家，凭什么"   ← 冒名顶替综合征
       ↓
  结果：内容一直在产出，但只在本地文件里
        永远在准备，永远没发布
```

这三个想法组成了一个死循环——你越不发布，越觉得要"准备够了再发"；越觉得没准备好，越不敢发。

打破它的方式不是"先变完美再发布"，是**先用一次不完美的发布打碎恐惧**。

### 四步破除方案

#### 第一步：降级发布目标

脑子里可能想的是"发表一篇被雪球推荐、被公众号转发的爆款文章"——目标太大，压得自己不敢动。

换个定义：

```
"发布" = 点一下"发布"按钮
       ≠ 获得 XXX 阅读量
       ≠ 被 XXX 认可
       ≠ 写得完美

本周目标：发布一篇。字数不限。只要点"发布"就算赢。
```

#### 第二步：从已有素材里挑第一篇

不需要新写什么。已有素材足够：

| 已有素材 | 适合平台 | 难度 |
|------|------|:---:|
| 某一天 A 股复盘（隐私脱敏后） | 雪球 | 最低 |
| AI 编程模型性价比排名 | 少数派 / 知乎 | 低 |
| 宽基 ETF 做T 策略 | 雪球 / 公众号 | 中 |
| 今日复盘中的"学到了什么" | 微博 / 即刻 | 最低 |
| 踏空焦虑的 24 小时反转 | 雪球 / 公众号 | 低 |

> 🔑 第一篇建议选"踏空焦虑的24小时反转"——有事件弧线（6/22暴涨→6/23暴跌）、有情绪张力（害怕踏空→幸好没追）、有数据支撑（有色两日+3.93%→-8.36%）。这种有叙事弧线的文章比纯分析更容易被传播。

#### 第三步：用 AI 降低打磨成本

你已经用 AI 辅助写复盘的——发布就是多做一步：

```
已有复盘文章（已写好）
  → Claude：改成适合雪球发布的版本，语气平实，有个人经历感
  → 你：隐私脱敏（去掉持仓数据、具体金额）
  → Claude：检查有没有让人看了不舒服的地方
  → 你：花 5 分钟改 → 发布
```

从"写一篇完美文章"变成"改一篇现成素材"——心理负担降低 90%。

#### 第四步：设定一个不可能失败的目标

```
第 1 周：在雪球发 1 篇（任何内容，只要点了发布）
第 2 周：发第 2 篇
第 3 周：发第 3 篇
...
第 12 周：回头看，已发 12 篇

3 个月后可能出现的情况：
├── 有人读，有人关注你
├── 有人质疑——好事，说明有人看
├── 有人私信说"你写的对我有帮助"
│   └── 这就是你前 10 个付费用户的来源
└── 发现：恐惧只是纸老虎——80% 在点"发布"那一刻消失
```

### 现在就行动

把这句话在心里过一遍：

> "本周五之前，我会在雪球发布一篇基于今日复盘的帖子。内容已经写好了。我只需要脱敏 + AI 润色 + 点发布。不需要完美。发布就是胜利。"

不是"有一天我要开始写公众号"——是**具体的日期、具体的平台、具体的行动**。

## 快速迭代改变自己

> 自我认知不是终点——改变才是。这一节专门回答：知道了问题之后，怎么高效地改。

### 为什么大多数自我改变失败

不是缺决心，是缺**反馈速度**。

```
传统模式（注定失败）：

下定决心 → 默默努力 → 很久以后 → 检查结果
              ↑                        ↑
         没有外部反馈              发现早已偏航

问题：反馈周期太长。从"我想变"到"我知道变了没"之间隔了几周甚至几个月。
没有中间反馈的自我改变 = 在黑暗中射箭，不知道有没有射中靶子。
```

### 核心机制：压缩反馈循环

> 引自 [[方法论]]：PDCA 的核心不是"计划做得完美"，而是**让 Plan→Do→Check→Act 转得足够快**。

```
慢速 PDCA（月级别）：
  Plan ──→ Do ──→ Check ──→ Act
   ↑                           │
   └──── 一个月后才转回来 ─────┘
  
  适合：年度目标（交易系统迭代、知识库建设）

快速 PDCA（天级别）：
  Plan → Do → Check → Act
   ↑                  │
   └── 当天就转回来 ──┘
  
  适合：行为习惯（公开发布、运动、做T纪律）
```

**一个人改变的速度 = PDCA 循环转一圈的时间。** 转得越快，进化越快。

### 五个加速器

#### 加速器一：把目标缩小到可以今天验证

| 原来的目标 | 缩小后 |
|------|------|
| "我要成为交易高手" | "今天做T一笔，盈亏都记下来" |
| "我要打造个人品牌" | "今天发一篇雪球帖子" |
| "我要练出好身材" | "今天打一小时羽毛球" |
| "我要学会量化交易" | "今天跑通一个回测脚本" |
| "我要提升认知深度" | "今天写 200 字复盘心得" |

> 🔑 目标大到需要一个月才验证 → 一个月后才发现没动 → 沮丧 → 放弃。目标小到今天就能完成 → 今天完成 → 有成就感 → 明天继续。**动量来自小胜，不是大愿。**

#### 加速器二：用外部反馈替代自我感觉

你自己觉得"今天我认真复盘了"——没用。需要外部反馈：

```
内部反馈（模糊、可自欺欺人）：
  "我感觉今天盯盘很专注"  ← 没有指标
  "我觉得这篇文章写得不错" ← 没有数据

外部反馈（具体、无法自欺欺人）：
  "今天这篇雪球帖子有 0 个评论" ← 真实数据，立刻知道问题在哪
  "这个月交易胜率 42%，盈亏比 1.1" ← 数字不会骗人
  "过去七天有三天没打羽毛球"   ← 事实就是事实
```

**没有外部反馈的自我认知 = 自嗨。** 把一切能数字化的数字化。

#### 加速器三：公开承诺

你已经在体验了——"公开发布是卡点"这个判断本身，就是在对话中说出来之后才变得无法回避的。

```
私下目标：只有你自己知道 → 放弃零成本 → 大概率半途而废
公开承诺：有人知道你在做什么 → 放弃有社会成本 → 完成概率高 3 倍
```

具体做法：
- 在雪球/公众号发布第一篇——之后你就有"读者期待"这个外部约束
- 在月度审计里写入"下月行动"——下月自己检查是不是做到了
- 跟一个信得过的朋友说："我每周二晚上发复盘，没发你骂我"

#### 加速器四：环境设计 > 意志力

不要试图用意志力对抗自己。改变环境，让正确的事变得容易，错误的事变得困难：

```
想多写文章？
  → 手机首屏放雪球/公众号编辑器，打开就能写
  → 不要放在"等我有整块时间再写"——永远不会有整块时间

想少刷短视频？
  → 把抖音从手机删除   （不是"控制自己少看"，是让自己看不到）

想坚持运动？
  → 球拍放在门口       （让"出门打球"的摩擦力降到零）
  → 加入一个固定局     （每周六上午10点，不去有人催）

想做T不冲动？
  → 条件单代替手动      （用系统替代情绪）
```

> 🔑 多数人把失败归因于"自己不够自律"。其实是不够聪明——**聪明人设计环境，笨人消耗意志力。**

#### 加速器五：只盯一个指标

同时想改五个方面 = 五个都改不好。每个阶段只盯一个：

```
当前阶段（6-7月）：
  核心指标：雪球发布篇数 → 目标：每周 ≥ 1 篇
  其他一切（交易胜率、体重、阅读量）→ 保持即可，不追求提升

下一阶段（8-9月）：
  核心指标：交易胜率 → 目标 ≥ 55%
  其他一切 → 保持即可
```

**一段时间只做一件事，做到自动化的程度，再换下一件。** 这就是快速迭代的真正含义——不是同时变快五个方面，是一个一个快速切换跑道。

### 你自己的加速方案

基于当前情况，未来 30 天的核心循环：

```
Plan：本周雪球发 1 篇
  Do：周四把复盘改成可发布版本 → 周五发布
Check：发布后 48 小时内——有没有评论？有没有阅读量？
  Act：有反馈 → 下一篇调整选题和写法
       没反馈 → 下一篇换平台或换主题

第二周：基于上周反馈，发第 2 篇
第三周：基于两周反馈，发第 3 篇
第四周：回头看——4 篇帖子，至少有一篇有反馈。
        这个反馈就是下一步的燃料。
```

> 一个月后你就会发现：改变不是来自"想通了什么"，是来自**做了、被反馈了、改了、又做了。**
