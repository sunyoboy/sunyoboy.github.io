## 核心原则：交易只在收盘后发生

```
上班族的正确交易节奏：

08:00-09:00  看一眼美股收盘 + 确认今天挂的条件单是否有效 → 5分钟
09:25        看一眼开盘（只看是否有极端事件触发条件单） → 30秒
09:30-15:00  上班。不看盘。不操作。不内耗。           → 0分钟
12:00-13:00  午休。可以看一眼指数涨跌（"哦，科创又涨了"）→ 可选，但绝不打开账户
15:00-15:30  跑脚本 → 看框架输出 → 决定明天怎么做     → 30分钟
收盘后       挂条件单 → 明天自动执行                   → 2分钟

每天总耗时 < 40分钟
```

| 上班族绝对不做的事 | 原因 |
|------|------|
| 盘中打开交易APP | 分时图劫持系统1 → 高潮点追涨 / 恐慌点割肉 |
| 厕所/午休偷偷看盘 | 看一秒就会看十分钟，看十分钟就会操作 |
| 用微信接收行情推送 | 注意力残留 → 下午工作全废 |
| 收盘后才"感觉"该买/卖 | 感觉 = 没有经过框架过滤 = 在情绪中做决策 |
| 盘中临时起意下单 | 没有止损、没有仓位计算、没有事件检查 |

## 每日时间线

### 盘前 · 5分钟（出门前）

```
□ 看一眼美股收盘（只盯三个：纳指/标普/美元指数）
□ 确认今天的条件单是否仍然有效
  例：天山 11.80 减仓单还在吗？在。
      中证500 8.88 限价单还在吗？在。
      不需要改 → 关掉APP → 出门上班
```

### 盘中 · 0分钟（办公室）

```
不打开任何交易软件。

如果收到了条件单触发通知：
  → 看一眼 → 确认成交 → 关掉
  → 不因为"赚钱了"高兴一整天
  → 不因为"亏钱了"毁掉一整天
```

### 午休 · 可选 5分钟

```
可以看一眼指数涨跌（"哦，科创50又涨了3%"）。
绝对不做：打开账户看盈亏、临时起意下单。
```

### 收盘后 · 30分钟（回家/下班路上）

```
15:05 跑 python3 scripts/discipline-check.py --portfolio
      → 成交额情绪？振幅？波动率？

15:10 跑 python3 scripts/ma5-deviation.py
      → 哪些标的触发信号？

15:15 做两个决定：
      ① 明天挂什么条件单？（入场/止损/仓位）
      ② 现有的条件单要不要改？

15:20 挂条件单。关APP。
      今天的工作结束了。明天你只需要等条件单触发。
```

### 晚间 · 不碰账户

```
21:30 看一眼明日计划（CLAUDE.md 晚间清单）
      不打开账户，不打开行情软件。
      看账户影响睡眠 → 影响明天判断 → 连环打击。
```

## 为什么上班族反而有优势

```
全职盯盘的人：
  每天 4 小时盯盘 → 看到 100 个"机会" → 操作 10 次
  → 其中 7 次是噪声 → 被手续费和情绪磨损消耗

上班族：
  每天 30 分钟收盘后决策 → 只看到 3 个信号
  → 只有框架说绿灯才操作 → 每周 ≤ 2 次
  → 不赚钱的交易日也赚了工资
```

**数据佐证**：上交所 5,300 万账户数据——交易最频繁的 20% 账户年化回报最低。

> 上班族天然被限制了交易频率。这不是劣势——是制度性的保护。你的老板在给你发工资的同时，也在帮你管住手。

## 上班族专用系统设计

### 设计原则

```
1. 所有决策在收盘后做完
2. 盘中只做一件事：等条件单触发
3. 你今天在办公室做的事，和你明天账户里发生的事，是两件互不干扰的事
4. 条件单 = 你的分身 → 替你盯盘、替你下单、不被情绪影响
```

### 条件单清单（每日收盘后维护）

| 类型 | 例 | 有效期 |
|------|------|:---:|
| 减仓/清仓 | 天山铝业 ≤ 11.80 → 卖出 350股 | 直到触发或手动撤单 |
| 低吸限价单 | 中证500ETF 8.88 → 买入 1,200份 | 当天有效 |
| 止损单 | 中证500 跌到 8.65 → 卖出 | 入场后立即挂 |

> 🔑 每天收盘后唯一的操作 = 校准这三个单子。其他时间不看盘。

### 离合器三项（收盘后嵌入）

```
在复盘模板"明日计划"第一行加：

  □ 今日条件单：入场___止损___仓位___（已挂/不挂/无操作）
  □ 天山铝业减仓监控：今日价格___（ < 11.80？→ 挂卖出条件单）
  □ PCE/事件日历：今日有事件吗？→ 有则延迟操作

这三个填空不做完，不复盘。不做完，不关电脑。
```

              你的框架在替你上班
