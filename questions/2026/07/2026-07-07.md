## 二、工具建设

### 决策：创建项目专属 Review Code Skill

**需求**：需要一个了解 KnowingDoing 项目规范的代码审查技能。

**审查维度设计**：

| 维度 | 检查内容 |
|------|------|
| **构建验证** | `npm run build` (VitePress) 是否通过 |
| **联动更新** | 新增文档 → catalog.md + config.mjs sidebar 是否同步 |
| **命名规范** | 文件名是否符合 `YYYY-MM-DD[-主题].md` 格式 |
| **文档结构** | 目录 + Frontmatter + 必要章节是否完整 |
| **脚本完整性** | Python 脚本的抓取函数、BUY_ZONES 配置是否最新 |
| **Git 规范** | commit message 格式、双 remote 状态 |

> 区别于 superpowers 的 requesting-code-review（流程）和 receiving-code-review（心态），本 Skill 聚焦于**执行审查的具体检查项**，与该项目的 CLAUDE.md 规范强绑定。
