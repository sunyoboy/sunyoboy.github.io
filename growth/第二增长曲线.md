## 为什么需要第二曲线

**核心矛盾**：第一曲线（A 股交易）目前还不能稳定盈利，而生活开支即将因孩子大幅增加。

```
当前状态：
├── 本金：~28 万，现金 24 万（85%）
├── 多数持仓浮亏（-15% ~ -51%）
├── 收入完全依赖市场行情，极不稳定
├── 配偶孕早期（预产期 2026/2），异地医保，保胎中
└── 即将新增育儿开支

五年目标（solo-dev-strategy.md）：
└── 2028-2029：被动收入 > 工资收入
```

第二曲线的目的不是"放弃交易"——是**在市场之外建立独立于行情的收入**，让你在市场不好时仍有现金流。

## 先排除：硬件维修

| 维度 | 现有技能 | 硬件维修 | 交集 |
|------|------|------|:---:|
| 核心能力 | Python、数据分析、AI、写作 | 电路焊接、芯片级诊断、物理拆装 | **零** |
| 收入模式 | 可规模化（代码/策略复用） | 计件制（修一台赚一台） | **零** |
| 地理约束 | 无（线上即可） | 强（必须在物理设备旁） | **零** |
| 学习曲线 | 2-3年 → 熟练 | 2-3年 → 刚入门 | — |
| 与飞轮关系 | — | 无法协同 | **零** |

你在北京、配偶孕早期需要随时照顾，不可能去学一门需要物理在场、计件赚钱、与现有能力无关的手艺。**这不叫第二曲线，叫重新开始。**

## 现有飞轮

你已经在 `solo-dev-strategy.md` 里写清楚了：

```
AI × 投资 × Python × 写作

每一项都在强化其他三项：
├── AI → 提高 Python 开发效率
├── Python → 量化回测 + 自动抓数据
├── 投资 → 交易经验是内容原材料
├── 写作 → 倒逼思考深度 + 建立个人品牌
└── AI + 写作 → 内容生产效率远超同行
```

第二曲线不是另起一行——是**把飞轮中尚未货币化的能力变成收入**。

## 三条路径

### 路径一：量化策略 / 交易工具

**契合度：★★★★★**

| 维度 | 评估 |
|------|------|
| 现有基础 | 已调研 QMT/MiniQMT/聚宽/TradingView，Python 熟练 |
| 已在做的事 | `fetch-market-data.py` 自动抓数据、条件单做T |
| 学习成本 | 低——技能树已有，只需产品化 |
| 收入模式 | 策略上架销售、信号订阅、定制脚本接单 |
| 与交易协同 | 写策略 → 提升交易水平 → 交易赚钱 + 策略卖钱，双向增强 |

**具体方向**：

| 产品 | 平台 | 定价模型 | 启动难度 |
|------|------|------|:---:|
| Pine Script 策略脚本 | TradingView | $9.99-29.99/月 订阅 | ⭐ |
| 量化回测模板 | GitHub + 文档 | 免费开源 → consulting | ⭐⭐ |
| 条件单/做T自动化 | 开源 + 定制 | 一次性 ¥99-299 | ⭐⭐ |
| QMT 实盘策略 | 券商平台 | 订阅制 | ⭐⭐⭐ |

### 路径二：AI 增强的付费内容

**契合度：★★★★☆**

| 维度 | 评估 |
|------|------|
| 现有基础 | 每日复盘已成习惯，AI 工具链（Claude/GPT/DeepSeek）熟练 |
| 已在做的事 | 每交易日写复盘，自动抓数据 + AI 分析 |
| 学习成本 | 极低——原材料已在产出，只需公开发布 |
| 收入模式 | 知识星球/付费 newsletter/小报童/少数派专栏 |
| 与交易协同 | 公开输出 → 倒逼思考质量 → 提升交易水平 |

**具体方向**：

| 产品 | 平台 | 定价 | 目标人群 |
|------|------|:---:|------|
| A 股每日复盘 | 知识星球 / 公众号 | ¥199/年 | 散户投资者 |
| ETF 做T 策略 | 付费专栏 + 实盘信号 | ¥299/年 | 想做T但不会的散户 |
| AI 辅助投资工具包 | Notion模板 / 自动化脚本 | ¥49-199 | 愿意用工具的投资者 |
| 个人知识管理方法论 | 小报童 / 少数派 | ¥39-99 | 想做 PKM 的人 |

**收入估算**：

```
知识星球 ¥199/年 × 100人 = ¥19,900/年
                × 300人 = ¥59,700/年
                × 500人 = ¥99,500/年
```

> 你每天写的复盘，只要公开发出去，就是资产的种子。差的不是能力，是「公开发布」这一步。

### 路径三：独立开发 / SaaS

**契合度：★★★☆☆**

| 维度 | 评估 |
|------|------|
| 现有基础 | wechat-publish.py、Chart.js、VitePress 建站 |
| 已在做的事 | 知识库站点、自动抓数据脚本 |
| 学习成本 | 中等——API 开发/用户系统/支付接入 |
| 收入模式 | SaaS 订阅、一次性购买 + 年费 |
| 当前阻碍 | 配偶孕期 + 双城往返，大块连续时间不足 |

**如果要做**，从最懂的需求切入：

| 方向 | 说明 | 为什么你适合 |
|------|------|------|
| 孕期管理工具 | HCG 追踪、产检日历、医保报销计算器 | 正在经历，最懂痛点 |
| A 股复盘自动化 | 抓数据→AI 分析→生成报告→推送 | 已经在用，产品化即可 |
