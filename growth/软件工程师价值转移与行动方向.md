# 软件工程师价值转移与行动方向

> 2026-07-08 · 在经济保守+AI突进的时代，成就感从「大厂螺丝钉」转向「直接创造可见价值」
>
> 原文作者：曼波哥（知乎）

---

## 一、价值转移的核心

```
过去: 大厂横跳 → 庞大系统螺丝钉 → APP日活成就感
现在: 直接创造 → 掌握主导权 → 用技术解决真实痛点

最大的成就感 = 掌控感
→ 不依赖庞大组织，凭自己的技术和一台电脑就能创造价值
```

---

## 二、五个方向与我的飞轮对照

### 方向 × 飞轮

| 方向 | 对应飞轮 | 当前进度 | 差距 |
|------|------|:--:|------|
| **AI+垂直领域** | AI 工作流 | 🔴 0% | 已有华泰Skill可起步 |
| 微型SaaS | 量化交易 | 🔴 0% | 选股系统就是SaaS雏形 |
| 传统产业数字化 | 无 | — | 不在规划中 |
| 开源/开发者工具 | Python 深度 | 🟡 30% | site已开源 |
| 知识输出 | 写作输出 | 🔴 0% | 每月1篇一直没写 |

### 最值得做的：AI+投资垂直领域

> 华泰Skill选股系统 + AI辅助复盘 = 本身就是「AI+投资」的垂直应用。打磨成小工具，比从头想新SaaS靠谱得多。

---

## 三、行动建议

| # | 行动 | 时间 | 产出 |
|:--:|------|------|------|
| 1 | 把现有复盘+选股流程自动化 | 7-8月 | 一键生成每日复盘Markdown |
| 2 | 用华泰Skill跑通选股→诊断→模拟交易全链路 | 8-9月 | 量化雏形 |
| 3 | 写一篇「VitePress个人投研系统搭建」 | 7月 | 知乎/掘金发布 |
| 4 | 开源选股脚本 | 9月 | GitHub Star |

### 三条原则

```
1. 转变心态: 「写代码的」→「用技术解决问题的」
2. 保持低成本: 不辞职创业·周末构建副业·验证后再扩
3. 保持技术敏锐: 熟练掌握AI编程工具(Cursor/Copilot/Claude Code)
```
